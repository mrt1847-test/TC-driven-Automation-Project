# Windows Installer Guide

Last aligned: 2026-06-01

This guide documents the baseline Windows packaging path for the Electron
desktop app. It produces an unsigned NSIS installer suitable for local QA and
internal distribution checks.

## Scope

The baseline installer includes:

- Electron main, preload, and renderer bundles from `apps/desktop/out`
- the FastAPI worker source under Electron `resources/worker`
- an NSIS installer generated by `electron-builder`

The baseline installer does not include:

- Python runtime bundling
- Worker Python dependency installation
- code signing
- notarization
- auto-update or release channels

## Prerequisites

Build from Windows.

```powershell
node --version
npm --version
python --version
```

Required versions:

- Node.js 20+
- Python 3.11+
- npm access to download `electron-builder@26.0.12` when `dist:win` is first run

Before launching the installed app, the machine must have a `python` command
available on `PATH` and the Worker dependencies installed for that Python
environment:

```powershell
npm run install:worker
```

The packaged Electron app starts the Worker from `resources/worker` with:

```text
python -m uvicorn worker.main:app --host 127.0.0.1 --port 8765
```

## Build Commands

Build the desktop bundles only:

```powershell
npm run build
```

Create an unpacked Windows app directory for quick inspection:

```powershell
npm run pack:win
```

Create the NSIS installer:

```powershell
npm run dist:win
```

Workspace-local equivalents:

```powershell
npm run pack:win --workspace=apps/desktop
npm run dist:win --workspace=apps/desktop
```

## Output

Artifacts are written under:

```text
apps/desktop/release/
```

Expected baseline outputs:

```text
apps/desktop/release/win-unpacked/
apps/desktop/release/TC Automation Studio-0.1.0-win-x64.exe
```

The exact installer file name follows `artifactName` in
`apps/desktop/electron-builder.json`.

## Configuration

Packaging config lives at:

```text
apps/desktop/electron-builder.json
```

Important baseline choices:

- `asar: true` packages desktop JavaScript bundles.
- `win.target: nsis` creates a Windows installer.
- `nsis.oneClick: false` enables an installer wizard.
- `nsis.perMachine: false` uses per-user installation.
- `extraResources` copies the Worker source and `requirements.txt`, while
  excluding tests and Python caches.

## Verification

After `npm run dist:win`:

1. Confirm `apps/desktop/release/TC Automation Studio-0.1.0-win-x64.exe` exists.
2. Run the installer on a Windows test machine.
3. Launch TC Automation Studio.
4. Open Settings and run validation.
5. Confirm the Worker health panel can reach `http://127.0.0.1:8765`.

If the Worker does not start, confirm Python is on `PATH` and run:

```powershell
python -m uvicorn worker.main:app --host 127.0.0.1 --port 8765
```

from the packaged `resources/worker` directory to inspect the error directly.

## Signing And Release Limitations

Current baseline artifacts are unsigned. Windows SmartScreen warnings are
expected for external distribution.

Out of scope for this baseline:

- EV/OV code signing certificates
- timestamp server configuration
- auto-update metadata
- release-channel publishing
- bundled Python runtime or virtual environment

Those should be added before public distribution.
