## Fundamental risk audit (2026-06-11)

Scope checked: Worker API trust boundaries, Project IDE file APIs, import/generation identity paths, Webwright/runner subprocess lifecycle, WSL command construction, and generated runtime state. Verification during audit: `python -m pytest tests --ignore=tests/e2e -q` passed from `apps/worker` (141 passed), and `npm run build` passed.

- **Critical — localhost Worker trust boundary + generated-file path traversal:** Worker currently exposes unauthenticated localhost APIs with wildcard CORS, while generated-file write/create/delete/rename paths do not consistently resolve and enforce containment under the generated project root. A browser page or compromised renderer path could mutate files outside `generated/`. Track both containment implementation (**C11-05**) and localhost API trust boundary hardening (**G-06**).
- **Resolved — Stop buttons now terminate tracked Worker subprocesses:** Webwright subprocess cancellation is resolved by **C3-09** and execution `runner.cli` cancellation by **C9-08**. Run/job ID uniqueness and same-second artifact collision hardening remains tracked separately as **C9-09**.
- **High — `automation_key` is not a project-unique identity:** imports accept explicit duplicate keys, the DB has only a non-unique index, and generated files/mappings/export rows are keyed by `automation_key`. Duplicate keys can silently overwrite generated flow/test files and merge incorrect mapping/export identities. Track as **C1-10**.
- **Resolved — POM method identity and page segmentation:** same-named steps no longer share a single `GeneratedPage` method; C7-15 scopes generated method names by case/step and repairs legacy shared POMs during selected raw refresh, and C7-17 segments generated page files by route evidence.
- **Medium — WSL Webwright command construction is shell-string based:** WSL mode builds a `bash -lc` string with `cd {webwright_root}` and manually quoted args. Paths/prompts/config values containing spaces, quotes, or shell metacharacters can fail or execute unintended shell syntax. Track as **C3-10**.
- **Medium — run/job artifact identifiers are collision-prone under fast or concurrent starts:** Webwright and execution run directories use second-resolution timestamps, while WebSocket job IDs are project-scoped for normal runs. Concurrent starts can mix logs/artifacts or overwrite run directories. Track as **C9-09** and include Webwright output roots in the verification.
